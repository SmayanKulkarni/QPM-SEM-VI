--- Slide 11 ---

Discounted Returns — Deep Dive
Sutton & Barto §3.3
Module 3 · Full Reinforcement Learning  |  Sutton & Barto
11
Recursive form (key identity for DP & TD):   Gₜ = Rₜ₊₁ + γGₜ₊₁
γ < 1 guarantees convergence
If rewards are bounded, the infinite sum is finite. This keeps value estimates stable throughout learning.
γ = 0 : Myopic agent
Agent only maximises Rₜ₊₁. Ignores all future. Works only if actions have no future consequences.
γ → 1 : Farsighted agent
Treats distant future rewards almost as valuable as immediate ones. Required for long-horizon games and robotics.
Intuition: present value of money
£100 today > £100 in a year. γ encodes how much the agent "discounts" future rewards relative to current ones.

Unified Notation for Episodic & Continuing Tasks
Sutton & Barto §3.4
Module 3 · Full Reinforcement Learning  |  Sutton & Barto
17
We need a single notation that covers BOTH task types
Episodic  (finite T)
Gₜ = Σ γᵏ Rₜ₊ₖ₊₁  (k=0 to T-t-1)
γ = 1 allowed when T is finite
Continuing  (T = ∞)
Gₜ = Σ γᵏ Rₜ₊ₖ₊₁  (k=0 to ∞)
γ < 1 required for convergence
The Absorbing State Trick
To unify both types: model episode termination as entering a special absorbing state that only transitions to itself with R=0. Now both tasks use the exact same formula.
Unified:  Gₜ = Σ γᵏ Rₜ₊ₖ₊₁   (k=0 to ∞)   with  T=∞ or γ=1  (but NOT both simultaneously)

--- Slide 18 ---

The Markov Property
Foundation of RL State Representation
Module 3 · Full Reinforcement Learning  |  Sutton & Barto
18
"Future is independent of the past, given the present"
P(Sₜ₊₁ | Sₜ, Aₜ) = P(Sₜ₊₁ | S₀, A₀, …, Sₜ, Aₜ)
What it means
The next state Sₜ₊₁ depends ONLY on (Sₜ, Aₜ). The full history before t is irrelevant if Sₜ summarises everything needed.
Why it matters for RL
Allows algorithms to use one-step transitions for learning. No need to store and process the full history of past states.
Conditional Independence
Sₜ₊₁ ⊥ {S₀,A₀,...,Sₜ₋₁,Aₜ₋₁} | Sₜ, Aₜ. Simplifies both modelling and computation enormously.
Violations & Partial Observability
If the state doesn't capture enough info, the Markov property fails. → Leads to POMDPs (Partially Observable MDPs).

--- Slide 19 ---

Worked Example: Pole Balancing
Episodic vs Continuing Formulation
Module 3 · Full Reinforcement Learning  |  Sutton & Barto
19
Classic control task from Sutton & Barto — illustrates both task types
Task: Apply forces to a cart on a track to keep a hinged pole upright. Failure = pole tips past threshold angle OR cart leaves track.
Formulation 1: Episodic
• Each attempt to balance = 1 episode
• Episode ends when pole falls (failure)
• Reward = +1 for each time step without failure
• Return = total steps before failure
• Goal: maximise episode length
Formulation 2: Continuing
• Single continuous interaction (no resets)
• Reward = −1 on failure, 0 otherwise
• Return = Σ γᵏ Rₜ₊ₖ₊₁ with appropriate γ
• K = steps before failure; related to −γᴷ/(1−γ)
• Goal: minimise frequency of failures forever
Key Insight: Both formulations lead to the same optimal behaviour — balance as long as possible. They differ only in reward shaping and notation.

The chance the policy π chooses action a when in state s. All probabilities from s sum to 1.

Action value
The expected return for specifically taking action a from state s, then following π.
Σₐ  ×
Weighted sum
Multiply each Q-value by how likely that action is, then add them all up to get V.

Quiz Game Show - Example
In a quiz game show there are 10 levels, at each level one question is asked and if answered correctly a certain monetary reward based on the current level is given. Higher the level, tougher the question but higher the reward.
At each round of play, if the participant answers the quiz correctly then s/he wins the reward and also gets to decide whether to play at the next level or quit. If quit then the participant gets to keep all the rewards earned so far. At any round if participants failed to answer correctly then s/he looses “all” the rewards earned so far. The game stops at level 10. The goal is to decide on the actions to play or quit maximizing total rewards

--- Slide 47 ---

Quiz Game Show - Solution
States: {level1, level2, …, level10}

Actions: Play at next level or quit

Rewards: Play at level1, level2, …, level10 generates rewards $10, $50, $100, $500, $1000, $5000, $10000, $50000, $100000, $500000 with probability p = 0.99, 0.9, 0.8, …, 0.2, 0.1 respectively. The probability here is a the probability of giving correct answer in that level. At any level, the participant losses with probability (1- p) and losses all the rewards earned so far.

Bellman Equation for V_π
v_π(s) = Σₐ π(a|s) Σₛ' P(s'|s,a) [r + γ·v_π(s')]
Meaning: V at state s equals the expected immediate reward plus discounted value of the next state — summed over all actions (weighted by π) and all possible next states.
Bellman Equation for Q_π
q_π(s,a) = Σₛ' P(s'|s,a) [r + γ·Σₐ' π(a'|s')·q_π(s',a')]
Meaning: Q at (s,a) equals the expected reward for taking action a, plus discounted value of the next state — evaluated under policy π from that next state onward.
The V ↔ Q Circular Relationship
V_π(s) = Σₐ π(a|s) · Q_π(s,a)
↕
Q_π(s,a) = Σₛ' P(s'|s,a) [r + γ·V_π(s')]
V is a weighted average of Q  |  Q depends on V of the next state

--- Slide 16 ---

SOLUTION OF RECYCLING ROBOT PROBLEM
There are two kinds of nodes: state nodes and action nodes. There is a state node for each possible state (a large open circle labeled by the name of the state), and an action node for each state–action pair (a small solid circle labeled by the name of the action and connected by a line to the state node).
Starting in state s and taking action a moves you along the line from state node s to action node (s, a).
Then the environment responds with a transition to the next state’s node via one of the arrows leaving action node (s, a). Each arrow corresponds to a triple (s, s', a), where s' is the next state, and we label the arrow with the transition probability, p(s'|s, a), and the expected reward for that transition, r(s, a, s').
Note that the transition probabilities labeling the arrows leaving an action node always sum to 1.

--- Slide 17 ---

SOLUTION OF RECYCLING ROBOT PROBLEM
If the energy  level is high, then a period of active search can always be completed without risk of depleting the battery. A period of searching that begins with a high energy level leaves the energy level high with probability α and reduces it to low with probability 1−α.
On the other hand, a period of searching undertaken when  the  energy  level  is  low  leaves  it  low  with probability β and depletes the battery with probability 1−β.
In the latter case, the robot must be rescued, and the battery is then recharged back to high.
Each can collected by the robot counts as a unit reward, whereas a reward of −3 results whenever the robot has to be rescued.
