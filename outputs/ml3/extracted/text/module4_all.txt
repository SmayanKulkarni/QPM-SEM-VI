Dynamic Programming hi
Dynamic Programming (DP) refers to a collection of algorithms that can be used to compute policies to solve a MDP given a perfect model of the environment.
Perfect Model means:
transition probabilities (the likelihood of ending up in a given state when taking an action from another state) 
reward model (the likelihood of getting a particular reward for being in a particular state) are known for the entire environment.

Limitations of DP
Assumption of a perfect model that is often unavailable for most problems
Computational expense


However, useful because they serve as a theoretical foundation on which most advanced RL methods are based.

Dynamic Programming
The key idea behind DP is the use of value functions to organize and structure the search for good policies.
DP will be used to compute v* and/or q*. Once we have these optimal value functions, deriving an optimal policy is trivial.
DP algorithms are obtained by turning the previously shown Bellman equations into update rules for improving approximations of the desired functions.

--- Slide 8 ---

Dynamic Programming
Herein given the complete model and specifications of the environment (MDP), we can successfully find an optimal policy for the agent to follow. 
It contains two main steps:
Break the problem into subproblems and solve it
Solutions to subproblems are cached or stored for reuse to find overall optimal solution to the problem at hand

Find out how good an arbitrary policy is
Find out the optimal policy for the given MDP

Policy Evaluation Algorithm
Start with some random state value function v₀.
Iteratively update the expected value of each of the states in v₀ using the previously shown Bellman equation for state value functions and the policy we are trying to evaluate.
Every time we update all the state values, we consider that a new version of the value function (v₁, v₂, v₃, etc.).
As we keep changing the value function it will converge to vπ, or the actual state value function for our policy π.
Since we are updating our value function “iteratively” until it reaches vπ, this is called iterative policy evaluation (IPE).

Policy Improvement
The reason for computing the value function for a policy is to help find better policies.

Once we know the value function for a policy, how can improve on that policy?
Policy Evaluation  →  "How good is current π?"
        ↓
Policy Improvement →  "Can we make π better?"
        ↓
    Repeat until no improvement = Optimal Policy π*

--- Slide 27 ---

Policy Improvement
Consider the case of an agent being in some state s and having access to some policy π and its state value function vπ. π already tells the agent which action to select but could one of the other actions be better?

One way to verify this is to choose some other action a that is not what π would choose and then continue to follow π afterward.
If this leads to a greater expected value than the current expected value for the state — vπ(s) — this implies that it’s better to select this action when in s than the action that π currently says to select.
In other words, changing the policy to select this other action a instead of the action that the policy currently says to select would mark an improvement to the policy.

Bootstrap
When you estimate something based on another estimation, it is called bootstrapping.
Bootstrapping is used in dynamic programming (DP) in reinforcement learning. 

.

--- Slide 48 ---

Bootstrapping In DP
In DP methods such as value iteration and policy iteration, bootstrapping is used to update the value function of a state based on the estimated value of the successor states. 
This allows for the efficient calculation of the optimal value function and policy.

Model-based Methods:
Utilize a function that predicts state transitions and rewards, allowing the agent to plan by thinking ahead.
Planning ahead enables the agent to make informed decisions by considering a range of possible choices.
Can substantially improve sample efficiency over model-free methods when a ground-truth model is available.
Challenges include the need to learn the model purely from experience, which can lead to biased models that result in sub-optimal behavior.
Model-free Methods:
Do not rely on a learned or ground-truth model of the environment.
Forego potential gains in sample efficiency that come with planning ahead.
Tend to be easier to implement and tune compared to model-based methods.

--- Slide 3 ---

While model-based methods can be more sample-efficient and capable of planning, they suffer from model bias and complexity. 

On the other hand, model-free methods are simpler and potentially more robust but require more interactions with the environment to learn effectively.

What is meant by Monte Carlo?
Monte Carlo methods require only experience — sample sequences of states, actions, and rewards from actual or simulated interaction with an environment. 
Monte Carlo methods in general refer to any technique that uses random sampling to estimate something. The name comes from the Monte Carlo casino in Monaco — because randomness and chance are central to the method.
The core idea: instead of computing something analytically, you let random experiments run and average the results.

Advantages of Monte Carlo over Dynamic Programing
Dynamic programming requires a complete knowledge of the environment or all possible transitions, whereas Monte Carlo methods work on a sampled state-action trajectory on one episode.
DP includes only a one-step transition, whereas MC goes all the way to the end of the episode to the terminal node.
One important fact about the MC method is that the estimates for each state are independent, which means the estimate for one state does not build upon the estimate of any other state, as in the case of DP.

Usable Ace
In the game of Blackjack, an "ace" card can have a value of 1 or 11, whichever is more advantageous for the player. 
A "usable ace" is an ace card that can be counted as 11 without causing the hand to bust. 
This means that the hand can still be played even if the player draws a card that would normally cause the hand to bust. 
For example, if a player has an ace and a 7 (totaling 18), they have a usable ace. If the player then draws a card with a value of 10 (e.g., a Jack or a Queen), their hand value becomes 28, but the ace can be counted as 1, so the hand value is reduced to 18, and the player can continue to play.

--- Slide 25 ---

Usable Ace
The term "usable ace" is important because it gives the player more flexibility in their play and can increase their chances of winning the game.
However, it is important to note that not all aces are usable aces, and the player must use their judgment to decide whether to count an ace as 1 or 11 based on the current state of the game.

--- Slide 26 ---

State value function computation for Blackjack Problem
Consider the policy that sticks if the player’s sum is 20 or 21, and otherwise hits.
To find the state-value function for this policy by a Monte Carlo approach, one simulates many blackjack games using the policy and averages the returns following each state.
Note that in this task the same state never recurs within one episode, so there is no difference between first-visit and every-visit MC methods.

--- Slide 27 ---

Solution of BlackJack Problem
It is straightforward to apply Monte Carlo to blackjack.
The episodes are all simulated games, so it is easy to arrange for exploring starts that include all possibilities.
In this case one simply picks the dealer’s cards, the player’s sum, and whether or not the player has a usable ace, all at random with equal probability.
As the initial policy we use the policy evaluated in the previous
blackjack example, that which sticks only on 20 or 21.
The initial action-value function can be zero for all state–action pairs.

MC estimation of Action-Values
When a model is not available, it's crucial to estimate action values rather than state values.
Without a model, state values alone are insufficient for determining a policy, so estimating action values becomes necessary.
The policy evaluation problem for action values is to estimate the expected return when starting in a particular state, taking a specific action, and then following a policy.
Monte Carlo methods for action values are similar to those for state values, with every-visit and first-visit methods being common.
The challenge arises when many relevant state-action pairs may never be visited during learning.
Maintaining exploration is crucial for learning accurate action values, ain to the exploration-exploitation trade-off in the n-armed bandit problem.

--- Slide 32 ---

MC estimation of Action-Values
One way to ensure exploration is the assumption of exploring starts, where the first step of each episode starts at a state-action pair with non-zero probability.
However, relying solely on exploring starts is not always practical, especially in real-world environments.
An alternative approach is to consider stochastic policies with non-zero probabilities of selecting all actions, ensuring exploration.
The assumption of exploring starts is sometimes useful but cannot be relied upon in general, especially in real-world scenarios.

--- Slide 37 ---

On-Policy
On-Policy learning Algorithms are the algorithms that evaluate and improve the same policy which is being used to select actions. 
That means we will try to evaluate and improve the same policy that the agent is already using for action selection. 
In short , [Target Policy == Behavior Policy]. Some examples of On-Policy algorithms are Policy Iteration, Value Iteration, Monte Carlo for On-Policy, Sarsa, etc.

--- Slide 38 ---

Off-policy learning
Target Policy pi(a|s): It is the policy that an agent is trying to learn i.e. agent is learning value function for this policy.


Behavior Policy b(a|s): It is the policy that is being used by an agent for action select i.e. agent follows this policy to interact with the environment.

--- Slide 39 ---


--- Slide 40 ---

Why use Off-Policy?
Some benefits of Off-Policy methods are as follows:
Continuous exploration: As an agent is learning other policy then it can be used for continuing exploration while learning optimal policy. Whereas On-Policy learns suboptimal policy.

Learning from Demonstration: Agent can learn from the demonstration.

Parallel Learning: This speeds up the convergence i.e. learning can be fast.

--- Slide 41 ---

On v/s off policy
The agent learns the value of the policy it is currently following. 
On-policy methods: The agent learns the value of the policy it is currently following.
Off-policy methods: The agent learns the value of a different policy (called the target policy) while following a separate behavior policy for exploration.
