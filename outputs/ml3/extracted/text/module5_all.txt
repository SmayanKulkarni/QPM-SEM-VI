Temporal Difference (TD) Learning in RL
Think of it this way: you're trying to learn how to predict rain. Three approaches:

Dynamic Programming (DP): You have a perfect weather model. You compute everything from equations.
Monte Carlo (MC): You wait for the entire day to end, then update based on whether it actually rained.
TD Learning: Mid-morning, you see dark clouds. You immediately update your prediction — without waiting for the day to end.

--- Slide 4 ---

TD learning is like learning to predict while doing—making small, smart adjustments instead of waiting for complete feedback. This makes it fast and practical for real-world tasks.
This means temporal difference takes a model-free. You can consider it learning from trial and error.

--- Slide 5 ---

The algorithm takes the benefits of both the Monte Carlo method and dynamic programming (DP) into account:


Like MC methods, TD methods can learn from experience without requiring a model of the environment’s dynamics.
Like DP methods, TD methods update estimates after every step based on other learned estimates without waiting for the outcome (this is called bootstrapping).

--- Slide 6 ---

Temporal difference (TD) learning, which is a model-free learning algorithm, has two important properties:
It doesn’t require the model dynamics to be known in advance
It can be applied for non-episodic tasks as well

Learning to play chess by playing full games and updating values based on wins/losses.

What is Temporal Difference Learning (focuses on the differences the agent experiences in time)
The key idea behind TD learning is that the agent can update its value estimates based on the difference between the current estimate and a better estimate, which is obtained by considering the immediate reward and the discounted value of the next state.
The basic TD learning update rule for the value function V(s) is:
V(s) <- V(s) + α [r + γV(s') - V(s)]
Where:
V(s) is the current estimate of the value of state s
r is the immediate reward received after taking an action in state s
γ is the discount factor (a value between 0 and 1)
V(s') is the estimate of the value of the next state s'
α is the learning rate (a value between 0 and 1)

--- Slide 10 ---

What is Temporal Difference Learning
Monte Carlo methods adjust their estimates only after the final outcome is known, but temporal difference methods tend to adjust predictions to match later, more accurate, predictions for the future, much before the final outcome is clear and know. 
This is essentially a type of bootstrapping.
Bootstrapping = Don't wait for the truth. Use your best current guess to improve your current guess — and keep refining.

--- Slide 11 ---

What is Temporal Difference Learning
Temporal difference learning in machine learning got its name from the way it uses changes, or differences, in predictions over successive time steps for the purpose of driving the learning process.
The prediction at any particular time step gets updated to bring it nearer to the prediction of the same quantity at the next time step

--- Slide 12 ---

Temporal Difference Learning Overview
TD learning introduces the concept of updating value estimates by bootstrapping - incorporating the current estimate to approximate the true value. 
This iterative process allows for continual refinement, striking a balance between Monte Carlo's reliance on complete episodes and dynamic programming's model-based approach.
Temporal Difference Methods vs. Monte Carlo and Dynamic Programming:
Unlike Monte Carlo, TD methods update estimates after each time step, offering quicker convergence. Additionally, TD methods do not require a model of the environment, differentiating them from dynamic programming.
TD methods have an advantage over DP methods in that they do not require a model of the environment, its reward, and next-state probability distributions, similarly TD methods have an advantage over Monte Carlo methods as they are naturally implemented in an online, fully incremental fashion. With Monte Carlo methods one must wait until the end of an episode because only then is the return known, whereas with TD methods one needs to wait for only one-time step.

--- Slide 13 ---

Temporal Difference methods are said to combine the sampling of Monte Carlo with the bootstrapping of DP, that is because in Monte Carlo methods target is an estimate because we do not know the actual expected value rather use a sample return from that particular episode, and in DP that target is an estimate because the value of the next state is not known instead the current estimate is used, and in TD the target is an estimate because of both the reasons, it samples the expected values and it uses the current estimate instead of the true state value.

Q-learning (off-policy TD Control)
In Off-policy learning, we evaluate target policy (π) while following another policy called behavior policy (μ) (this is like a robot following a video or agent learning based on experience gained by another agent).
In the Q-learning, target policy is a greedy policy and behavior policy is the Epsilon-greedy policy (this ensures exploration).

What is Q-Learning?
Q-learning is a model-free, value-based, off-policy algorithm that will find the best series of actions based on the agent's current state. The “Q” stands for quality. Quality represents how valuable the action is in maximizing future rewards.

--- Slide 32 ---

Q(s,a)="How much total reward will I get if I take action a in state s, and play optimally after?"

--- Slide 33 ---

Q-Table
The agent will use a Q-table to take the best possible action based on the expected reward for each state in the environment. In simple words, a Q-table is a data structure of sets of actions and states, and we use the Q-learning algorithm to update the values in the table.

SARSA (On-Policy)
SARSA (State-Action-Reward-State-Action) TD method to solve the Control problem, which means it tries to find an optimal policy for a given task.
In SARSA, the agent learns by updating its Q-value estimates based on the current policy, meaning that it uses the same policy for both action selection and value updates. 
Specifically, SARSA learns by estimating the value of taking an action in a given state and following the current policy thereafter.

--- Slide 40 ---

SARSA (On-Policy)
It updates the Q-value of the current state-action pair based on the current reward, the Q-value of the current state-action pair, and the estimated Q-value of that next state-action pair.
Because SARSA uses the same policy for both action selection and value updates, it is considered an on-policy method.

The goal is to eat the big pile of cheese at the bottom right-hand corner and avoid the poison. The episode ends if we eat the poison, eat the big pile of cheese, or if we take more than five steps.
 The learning rate is 0.1
The discount rate (gamma) is 0.99
The reward function goes like this:
+0: Going to a state with no cheese in it.
+1: Going to a state with a small cheese in it.
+10: Going to the state with the big pile of cheese.
