# Detailed Repository Structure

## Directory Reference

### `curriculum/` — Course Materials & Syllabuses

```
curriculum/
├── units/
│   ├── UNIT 1/
│   │   ├── A Overview of Portfolio Management/
│   │   ├── B Expected Risk and Returns/
│   │   └── UNIT 1 Portfolio Management.docx
│   ├── UNIT 2/
│   │   └── (unit 2 materials)
│   ├── UNIT 3/
│   │   ├── PART 1 Modelling Asset Returns/
│   │   └── PART 2 Simulation Approaches/
│   ├── UNIT 4/
│   ├── UNIT 5/
│   └── UNIT 6/

├── term-test-1-syllabus.pdf  # Term Test 1 scope
```

**Use**: Primary source for note generation and study material creation

### `tools/` — Automation Scripts & Workflows

Reusable scripts for content extraction, processing, and compilation.

#### `extraction/` — Content Extraction
```
tools/extraction/
├── extract_unit456.py              # Extract specific unit content
├── extract_and_prep.py             # Full extraction + preparation pipeline
├── build_complete_study_source.py  # Build complete source documents
└── (Processes PDFs, DOCX, PPTX → Markdown/JSON)
```

**Purpose**: Convert course materials into structured, processable format

#### `compilation/` — LaTeX Compilation
```
tools/compilation/
├── compile_latex.sh                # Bash script to compile LaTeX → PDF
├── README-compilation.md           # Detailed compilation procedures
└── (Converts .tex files to PDF with proper dependencies)
```

**Purpose**: Generate professional PDF study guides from LaTeX source

#### `processing/` — Post-Processing
```
tools/processing/
├── polish_tex_equations.py         # Clean up and format equations
└── reformat_latex.py               # Restructure LaTeX for readability
```

**Purpose**: Improve formatting and readability of extracted content

**Usage Pattern**:
1. Extraction scripts → produce raw content
2. Processing scripts → clean and format
3. Compilation scripts → generate final PDFs

## Workflow Data Flow

```
┌─────────────────┐
│  curriculum/    │        ← Source: Course materials
│  subjects/      │        ← Source: Specialized content
└────────┬────────┘
         │
         ├──→ extraction tools → Markdown/JSON
         │
         └──→ AI generation (trigger phrase)
                │
                ├→ study-materials/generated/exam-notes
                ├→ study-materials/generated/flashcards
                ├→ study-materials/generated/revision-guides
                │
                └──→ compilation tools (LaTeX)
                     │
                     └─→ outputs/compiled/ (PDFs)
```

---

## File Organization Principles

1. **Separation of Concerns**: Source materials, generation scripts, and outputs are distinct
2. **Subject-based**: New subjects can be added under `subjects/`
3. **Output Types**: Different output formats go to different folders
4. **Scriptability**: Tools are standalone and reusable
5. **Documentation**: Each tool and workflow has supporting docs

## Maintenance Notes

- **Outputs**: Clear `outputs/latex/` intermediates after PDF generation to save space
- **Study Materials**: Auto-generated files are safe to delete and regenerate
- **Source Files**: Keep `curriculum/` and `subjects/` pristine (read-only)
- **Tools**: Test scripts in isolated branches before committing
